# image-react — CLS-safe responsive `<Image>` on nifra

A nifra + React app demonstrating `@nifrajs/image` + `@nifrajs/web-react/image`: a **responsive `<Image>`**
with reserved layout space (no CLS), **lazy-by-default** loading, an **eager + high-priority LCP**
image, and a pluggable **CDN loader**.

```sh
bun run examples/image-react/build.ts
bun examples/image-react/server.ts        # http://localhost:3000
```

- `<Image src width height alt />` is required to carry intrinsic `width`/`height` — they reserve the
  box so the page doesn't shift when the image loads.
- The `loader` builds per-width URLs for a responsive `srcSet`; the demo uses a local stand-in
  (`loader.ts`) that the server serves as a **labeled SVG sized by `?w=`**, so the number you see is the
  candidate your browser actually picked. Swap it for `cloudflareLoader()` (from `@nifrajs/image`) in prod.
- The hero uses `priority` → `loading="eager"` + `fetchpriority="high"` (React 19 also preloads it).
  The below-the-fold images are `loading="lazy"` + `decoding="async"`.

## Self-hosted resize (`@nifrajs/image/server`)

The last image on the page is a **real** `1600×900` PNG (`public/photo.png`, generated by `build.ts`)
resized on the fly by nifra's own endpoint — no CDN. `server.ts` mounts it at `/_image`:

```ts
import { createImageHandler } from "@nifrajs/image/server"
const image = createImageHandler({ root: publicDir })   // local sources scoped to ./public
app.get("/_image", (c) => image(c.req))
```

`routes/index.tsx` points `<Image>` at it with `selfHostedLoader({ endpoint: "/_image" })`. Open
DevTools → Network and you'll see `/_image?src=%2Fphoto.png&w=…` return `image/webp` (negotiated from
`Accept`), `immutable`-cached with an `ETag`. The handler is SSRF- and DoS-hardened by default — try it:

```sh
curl -i 'http://localhost:3000/_image?src=%2Fphoto.png&w=400' -H 'Accept: image/webp'   # 200 image/webp
curl -i 'http://localhost:3000/_image?src=%2F..%2F..%2Fetc%2Fpasswd&w=10'                # 403 (traversal)
curl -i 'http://localhost:3000/_image?src=https%3A%2F%2Fevil.com%2Fx.png&w=10'           # 403 (not allowlisted)
```

The other images use a local stand-in **CDN** loader (`loader.ts`): the server's `/img/*` endpoint
serves a **labeled SVG sized by `?w=`**, so the number you see is the candidate your browser actually
picked. Swap it for `cloudflareLoader()` (from `@nifrajs/image`) in prod. The core bundles **no image
codec** — the loader is the seam, pointing at a CDN or nifra's own `@nifrajs/image/server` (Bun.Image).
